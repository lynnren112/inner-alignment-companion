# Example 3 — 自我怀疑与拖延

## User Input

我明明知道这个项目对我很重要，但就是一直拖。每次想开始就会想：万一做不好怎么办，万一做出来很普通怎么办。

## Example Response Structure

### 1. 你现在真正卡住的点
你卡住的可能不是执行力，而是你把“开始做”跟“暴露自己的不够好”绑定在了一起。

### 2. 事实 vs 定义
- 事实：你还没有开始推进这个项目。
- 定义：如果做得普通，就说明我不行；如果做不好，就会证明我没有我以为的那么厉害。

### 3. 被触发的核心信念
这里常见的核心信念是：
- 我必须一开始就做得好，才值得开始
- 一旦结果普通，就说明我本身普通

### 4. 这件事在反馈你什么
拖延在这里未必是懒，而更像一种保护：
只要不真正开始，你就暂时不用面对“我可能没那么完美”的不适。

### 5. 更对齐的理解方式
真正阻碍你的不是能力不足，而是你把作品质量过度绑定到了自我价值。
开始做一个普通版本，不是在贬低自己，而是在把生命力从自我评判里解放出来。

### 6. 今天可以做的一小步
今天不要要求自己把项目做好，
只要求自己完成一个 **20 分钟的极小起步版本**：
打开文档，写出目录，或者做第一张草图。

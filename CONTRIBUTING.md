# Contributing

Thanks for contributing to `inner-alignment-companion`.

## What kinds of contributions are welcome

- clearer README wording
- better examples
- stronger boundaries / safety guidance
- cleaner installation instructions
- reusable publishing-template improvements
- metadata cleanup in `SKILL.md`

## Before opening a PR

1. Read [`README.md`](README.md)
2. Read [`PUBLISHING-CHECKLIST.md`](PUBLISHING-CHECKLIST.md)
3. Keep `SKILL.md` as the single install entry
4. Prefer small, reviewable changes

## Contribution principles

- Preserve clarity over cleverness
- Preserve usability over theoretical completeness
- Keep the repo approachable for first-time visitors
- Do not turn the skill into therapy, crisis care, or destiny language
- Respect the repo's boundaries and safety framing

## If you change the public-facing experience

Please also check whether you should update:

- `CHANGELOG.md`
- `examples/`
- `references/README.md`
- `templates/`

## Good PR examples

- improve one install step
- add one strong example
- clarify one safety boundary
- improve one reusable template file

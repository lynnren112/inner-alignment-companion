# Publishing Checklist for a Public Hermes Skill Repo

Use this checklist before publishing your next skill repository.

## 1. Core identity

- [ ] Repo name matches the skill name
- [ ] `README.md` title matches the skill name
- [ ] `SKILL.md` frontmatter uses the correct `name:`
- [ ] `SKILL.md` description is concise and public-facing
- [ ] License is present

## 2. Installability

- [ ] `SKILL.md` is the single install entry
- [ ] README shows a minimal local install path
- [ ] README includes one copy-pasteable invocation example
- [ ] README explains what users should copy into Hermes

## 3. Public readability

- [ ] README has a short top summary
- [ ] README includes quick start
- [ ] README includes who it is for / not for
- [ ] README includes safety / boundaries
- [ ] README includes examples and references links

## 4. Productization

- [ ] Add at least 3 realistic examples
- [ ] Add a `references/README.md`
- [ ] Add a changelog entry for the release
- [ ] Add template files if this pattern should be reused
- [ ] Keep repo structure simple enough for first-time visitors

## 5. Release hygiene

- [ ] Check `git status`
- [ ] Review rendered README on GitHub
- [ ] Confirm internal links work
- [ ] Confirm install commands are still valid
- [ ] Tag or document the current version in `CHANGELOG.md`
